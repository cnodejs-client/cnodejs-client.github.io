# CNode Client Collection #

CNode客户端收藏

## 这个组织用来做什么? ##

CNode目前有非常多的客户端实现，但是目前并没有一个比较好的整理。

因此，这个组织被创建，用来收集各种CNode的客户端项目，通过fork的形式引用，方便查找。

“客户端”在这里不仅仅狭义的限定为Android或者iOS客户端，而是更为广义的定义，比如基于AngularJS或者VueJS的Web实现，也包含在内。

## 收集的标准是什么? ##

- 以CNode客户端为项目目标(而不是例如内容抓取爬虫或者用户数据分析这样的项目)

- 基于[CNode API](https://cnodejs.org/api)实现

- 有足够的完成度

- 开源并且项目托管于GitHub

- 最低限度的文档

- 可以被成功构建并运行

## 如何提交一个客户端项目? ##

如果你是一个CNode的客户端作者，或者你发现了一个CNode客户端项目，只要符合上面的要求，请在[这里](https://github.com/cnodejs-client/cnodejs-client.github.io/issues)通过创建一个issues的方式提交你的发现。

当确定符合要求之后，这个项目就会被fork到组织的项目列表中。

## 如何解决名字冲突问题? ##

项目名称可能会有冲突，目前使用GitHub的默认解决方案，即新fork的项目和之前的项目有冲突，则在新项目名称后面添加顺序序号。

因此，对于项目名称，实际上是执行先来后到的原则。即便项目名称被添加了序号，也并不是个大问题。

如果你不希望产生名称冲突，请起一个独一无二的名字。

## 如何同步项目? ##

组织中被fork的项目，其主要目的是为了创建一个引用，因此可能无法保证分支进度和原项目同步。

因此，在查看一个项目的时候，你应该返回原始项目去查看。

当然，如果你是原项目的作者，你可以申请加入组织来保持你自己的项目的更新进度。

## 之后的一些计划是什么? ##

目前并没有什么计划。

一个可能的想法是，邀请一些CNode客户端项目作者，分享一些CNode客户端开发上的技术细节和心得。

由于目前CNode客户端的各种实现，基本上包含了目前业界上主流的开发技术。

如果将这些内容整理出来并生成技术文章或者电子书的形式分享出来，可能会对一些想要去实践或者学习客户端开发的人，在技术选择上具有指导性意义。

上面的目前仅仅是一个想法而已，并没有明确的实施计划。

如果你有什么有建设性意义的想法，请提交一个[issues](https://github.com/cnodejs-client/cnodejs-client.github.io/issues)来讨论。
